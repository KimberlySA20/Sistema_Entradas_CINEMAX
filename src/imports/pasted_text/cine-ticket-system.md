🎬 Sistema Web de Compra de Entradas de Cine
1. Descripción del proyecto

El sistema es una aplicación web responsive que permite a los usuarios consultar películas disponibles en cartelera, preestrenos y próximos estrenos, seleccionar funciones disponibles, elegir asientos dentro de la sala y comprar entradas junto con snacks para retiro en el cine.

El sistema también incluye registro y autenticación de usuarios, permitiendo guardar información de compras y mejorar la experiencia del cliente.

La aplicación está diseñada para funcionar en dispositivos móviles y computadoras, ofreciendo una experiencia intuitiva e interactiva.

2. Objetivos del sistema
Objetivo general

Desarrollar una aplicación web interactiva que permita a los usuarios consultar películas, seleccionar funciones, elegir asientos, comprar snacks y gestionar su cuenta de usuario.

Objetivos específicos

Mostrar películas en cartelera, preestrenos y próximos estrenos.

Permitir el registro y autenticación de usuarios.

Mostrar horarios disponibles de cada película.

Permitir seleccionar asientos de manera interactiva.

Ofrecer compra anticipada de snacks.

Generar un resumen de compra antes de la confirmación.

Adaptar la interfaz a dispositivos móviles y escritorio.

3. Actores del sistema
Usuario visitante

Persona que accede al sistema sin haber iniciado sesión.

Puede:

consultar cartelera

ver próximos estrenos

ver horarios

registrarse

Usuario registrado

Usuario con cuenta en el sistema.

Puede:

iniciar sesión

comprar entradas

seleccionar asientos

comprar snacks

ver historial de compras

5. Requerimientos funcionales
RF01 – Visualización de cartelera

El sistema deberá mostrar las películas disponibles actualmente en cartelera.

RF02 – Visualización de preestrenos

El sistema deberá mostrar películas disponibles en preestreno.

RF03 – Visualización de próximos estrenos

El sistema deberá mostrar películas que aún no están disponibles pero que serán estrenadas próximamente.

Cada película deberá incluir:

título

fecha de estreno

sinopsis

imagen promocional

RF04 – Registro de usuario

El sistema deberá permitir a un usuario registrarse proporcionando:

nombre

correo electrónico

contraseña

RF05 – Inicio de sesión

El sistema deberá permitir a los usuarios autenticarse utilizando:

correo electrónico

contraseña

RF06 – Consulta de horarios

El sistema deberá mostrar los horarios disponibles para cada película.

Cada horario deberá incluir:

sala

idioma

formato (2D / 3D)

hora

RF07 – Selección de asientos

El sistema deberá mostrar un mapa interactivo de la sala de cine donde el usuario podrá seleccionar asientos disponibles.

Estados de asientos:

disponible

ocupado

seleccionado

RF08 – Compra de snacks

El sistema deberá permitir al usuario agregar snacks antes de confirmar la compra de entradas.

RF09 – Resumen de compra

El sistema deberá mostrar un resumen con:

película

horario

asientos seleccionados

snacks

total a pagar

RF10 – Confirmación de compra

El sistema deberá permitir confirmar la compra de las entradas.

RF11 – Historial de compras

El sistema deberá permitir a los usuarios registrados visualizar el historial de sus compras.

6. Requerimientos no funcionales
RNF01 – Diseño responsive

La aplicación deberá adaptarse correctamente a diferentes tamaños de pantalla.

RNF02 – Usabilidad

La aplicación deberá ofrecer una navegación clara y sencilla.

RNF03 – Interactividad

La selección de asientos deberá ser dinámica e interactiva.

RNF04 – Seguridad

Las contraseñas de los usuarios deberán almacenarse de forma segura utilizando técnicas de cifrado.

Como usuario

Como usuario
quiero comprar snacks antes de la función
para únicamente retirarlos al llegar al cine.

Como usuario

para confirmar que la información sea correcta.

Como usuario

para asegurar mis asientos.

HU10 – Ver historial de compras

Como usuario registrado
quiero ver mis compras anteriores
para consultar mis entradas adquiridas.
